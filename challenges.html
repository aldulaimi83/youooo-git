Hands-On Git Challenges

Mark each challenge complete after you finish it in a real repo.

1. Create Your First Repo

Create a folder, run git init, add a README, and make your first commit.

2. Stage and Commit Files

Edit two files, stage them, then save them with one commit.

3. Create a Feature Branch

Create a feature branch, make a change, and commit it there.

4. Push to GitHub

Connect a local repo to GitHub and push your code to the remote.

5. GitHub Pages Update

Make a change to a static site, commit it, and publish by pushing to main.

6. Resolve a Conflict

Create a merge conflict on purpose, fix it manually, and commit the resolution.

Workflow

Recommended Feature Branch Flow

Use this pattern for most practical work.

git switch main git pull origin main git switch -c feature-update git add . git commit -m "Build feature" git push -u origin feature-update
