Git Challenges

Complete hands-on tasks and mark them done to build confidence with real Git workflows.

1. Create Your First Repo

Create a folder, run git init, add a README, and make your first commit.

2. Track Two File Changes

Edit two files, stage them with git add ., and commit the update.

3. Build a Feature Branch

Create a new branch, change a page, commit, and merge it into main.

4. Push to GitHub

Connect a local repo to GitHub and push your branch with upstream tracking.

5. GitHub Pages Update

Update a static website repo and publish the changes by pushing to main.

6. Conflict Resolution Lab

Create a merge conflict on purpose, resolve it, then commit the fixed result.
