Git Lessons with Sidebar Navigation

Study lessons in sequence. Levels unlock as you progress.

Lesson Navigation

1. What is Git?

Git is a version control system. It tracks changes in files over time and helps you work safely with your own history and with teams.

2. Start a Repository

git init

This command creates a Git repository in the current folder.

3. Check Status

git status

This shows untracked files, modified files, staged files, and your current branch.

4. Stage Files

git add .

This stages your changes for the next commit.

5. Commit Changes

git commit -m "Initial commit"

A commit saves a snapshot of your staged work.

6. View History

git log --oneline

This gives you a readable short history of commits.

7. Branches

git switch -c feature-navbar

Create and switch to a new feature branch in one step.

8. Merge

git merge feature-navbar

Merge branch work into your current branch.

9. GitHub Pages Workflow

git add . git commit -m "Update site" git push origin main

This is a common workflow for updating a GitHub Pages website.

10. Git in VS Code

VS Code can stage changes, show diffs, write commit messages, and push updates through Source Control or terminal.

11. Stash and Rebase

git stash git rebase main

Stash saves temporary work. Rebase moves your branch commits on top of updated main.

12. Revert and Reset

git revert <commit-id> git reset --hard HEAD~1

Revert safely undoes with a new commit. Reset moves history back and can discard work.
