Core Lessons

Git Lessons

Study the lessons and mark each one complete to track your progress.

1. What is Git?

Git is a version control system that tracks file changes over time and helps teams collaborate safely.

2. Start a Repository

git init

Creates a new Git repository in your current folder.

3. Check Status

git status

Shows modified, staged, and untracked files plus the active branch.

4. Stage Files

git add .

Stages all changed files for the next commit.

5. Commit Work

git commit -m "Initial commit"

Saves a snapshot of your staged changes with a message.

6. Read History

git log --oneline

Shows a clean short history of previous commits.

1. Create Branches

git branch feature-login

Create a separate line of work without changing main.

2. Switch Branches

git switch feature-login

Move between branches safely.

3. Merge Changes

git merge feature-login

Bring branch changes into your current branch.

4. Add a Remote

git remote add origin https://github.com/username/repo.git

Connect your local repo to GitHub.

5. Push Code

git push -u origin main

Upload your local history to the remote repository.

6. Pull Updates

git pull origin main

Bring new remote changes into your local branch.

1. Rebase

git rebase main

Replay your branch commits on top of the latest main for a cleaner history.

2. Stash Work

git stash

Temporarily save uncommitted work.

3. Restore Stash

git stash pop

Bring the stashed work back.

4. Revert a Commit

git revert <commit-id>

Undo a previous commit by creating a new reversing commit.

5. Reset History

git reset --hard HEAD~1

Move history back and discard working changes. Use carefully.

6. Resolve Conflicts

git add .
git commit -m "Resolve conflict"

After fixing conflict markers manually, stage and commit the resolved files.

GitHub Team Flow

Feature Branch Workflow

This is a clean and common workflow for real projects.

Pull latest main
→
Create branch
→
Build feature
→
Commit work
→
Push branch
→
Merge to main

Step 1

git switch main
git pull origin main

Always start from the latest main branch.

Step 2

git switch -c feature-navbar

Create your feature branch for isolated work.

Step 3

git add .
git commit -m "Build navbar"

Save progress in small clear commits.

Step 4

git push -u origin feature-navbar

Push your feature branch to GitHub.

Step 5

git switch main
git merge feature-navbar

Merge completed work into main.

Step 6

git push origin main

Publish the updated main branch.
