Lessons

1. What is Git?

Git is a version control system. It helps you track changes in files, go back to earlier versions, and collaborate with other people.

2. Start a Git repository

git init

This creates a new Git repository in your project folder.

3. Check project status

git status

This shows changed files, staged files, and branch info.

4. Add files

git add .

This stages all changed files for the next commit.

5. Save changes

git commit -m "Initial commit"

This records a snapshot of your staged changes.

1. View commit history

git log --oneline

Shows a short list of previous commits.

2. Create a new branch

git branch feature-login

Creates a new branch for your work.

3. Switch branches

git switch feature-login

Moves you to another branch.

4. Merge a branch

git merge feature-login

Merges changes from one branch into the current branch.

5. Connect to GitHub

git remote add origin https://github.com/username/repo.git

Links your local project to a remote repository.

6. Push changes

git push -u origin main

Uploads your local commits to GitHub.

1. Rebase

git rebase main

Moves your branch changes on top of the latest main branch.

2. Stash temporary work

git stash

Saves uncommitted changes temporarily.

3. Restore stashed work

git stash pop

Brings back your saved temporary changes.

4. Undo a commit safely

git revert <commit-id>

Creates a new commit that undoes an earlier commit.

5. Reset branch

git reset --hard HEAD~1

Moves branch history back one commit and discards changes. Use carefully.

6. Resolve conflicts

Conflicts happen when Git cannot automatically combine changes. Edit the file, keep the correct code, then:

git add .
git commit -m "Resolve merge conflict"
