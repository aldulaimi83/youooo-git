Lessons

Beginner to Expert Lessons

Use the tabs below to move through the Git learning journey.

1. What is Git?

Git is a version control system that tracks changes in your files over time. It helps you save progress, return to earlier versions, and work with others.

2. Create a Repository

git init

This starts a new Git repository inside your project folder.

3. Check What Changed

git status

Shows untracked files, modified files, staged files, and your current branch.

4. Stage Files

git add .

Moves all changed files into the staging area so they can be committed.

5. Save a Snapshot

git commit -m "Initial commit"

Records a snapshot of the staged changes with a message.

6. View History

git log --oneline

Shows your commit history in a short readable format.

1. Create a Branch

git branch feature-navbar

Creates a separate line of development for new work.

2. Switch Branches

git switch feature-navbar

Moves you from your current branch to another branch.

3. Merge Work

git merge feature-navbar

Combines work from the selected branch into your current branch.

4. Connect to GitHub

git remote add origin https://github.com/username/repo.git

Links your local project to a remote repository.

5. Upload Changes

git push -u origin main

Pushes your local commits to GitHub and sets the upstream branch.

6. Download Changes

git pull origin main

Gets the latest changes from GitHub and merges them into your branch.

1. Rebase for Cleaner History

git rebase main

Replays your branch commits on top of the latest main branch.

2. Save Temporary Work

git stash

Stores your uncommitted changes so you can switch tasks safely.

3. Restore Stashed Changes

git stash pop

Applies the latest stashed changes back into your working directory.

4. Undo Safely

git revert <commit-id>

Creates a new commit that undoes an older commit without rewriting history.

5. Move Back in History

git reset --hard HEAD~1

Moves the branch backward and discards local changes. Use carefully.

6. Resolve Conflicts

git add .
git commit -m "Resolve conflict"

After editing conflicting files, add them and create a conflict resolution commit.

Workflow

Basic Git Branching Flow

This is the common pattern many teams use when building features.

main
→
create feature branch
→
make changes
→
commit work
→
push branch
→
merge to main

Step 1

Start from the latest main branch.

git switch main
git pull origin main

Step 2

Create your feature branch.

git switch -c feature-login

Step 3

Make edits and save them with commits.

git add .
git commit -m "Build login UI"

Step 4

Push your branch to GitHub.

git push -u origin feature-login

Step 5

Merge your work back after review.

git switch main
git merge feature-login

Step 6

Push the updated main branch.

git push origin main
